# Luban 时间类型使用文档

## 概述

Luban 新增了 5 种时间单位类型，用于更方便地配置时间相关的数值。这些类型会自动将输入值转换为毫秒（long 型）进行存储和导出。

## 新增类型

| 类型名称 | 说明 | 转换规则 |
|---------|------|---------|
| `day` | 天 | 输入值 × 24 × 60 × 60 × 1000 |
| `hour` | 小时 | 输入值 × 60 × 60 × 1000 |
| `minute` | 分钟 | 输入值 × 60 × 1000 |
| `second` | 秒 | 输入值 × 1000 |
| `millisecond` | 毫秒 | 输入值（不转换） |

## 使用方法

### 1. 在配置定义中使用

在 Bean 定义文件中，可以直接使用这些类型：

```xml
<bean name="TaskConfig">
    <var name="id" type="int"/>
    <var name="name" type="string"/>
    <var name="cooldown_time" type="second"/>      <!-- 冷却时间（秒） -->
    <var name="duration" type="minute"/>           <!-- 持续时间（分钟） -->
    <var name="expire_time" type="day"/>           <!-- 过期时间（天） -->
    <var name="delay" type="millisecond"/>         <!-- 延迟时间（毫秒） -->
</bean>
```

### 2. 在 Excel 中填写数据

在 Excel 配置表中，直接填写对应单位的数值即可：

| id | name | cooldown_time | duration | expire_time | delay |
|----|------|---------------|----------|-------------|-------|
| 1 | 任务1 | 30 | 5 | 7 | 500 |
| 2 | 任务2 | 60 | 10 | 14 | 1000 |

**说明**：
- `cooldown_time` 填写 `30`，实际存储为 `30000` 毫秒（30秒）
- `duration` 填写 `5`，实际存储为 `300000` 毫秒（5分钟）
- `expire_time` 填写 `7`，实际存储为 `604800000` 毫秒（7天）
- `delay` 填写 `500`，实际存储为 `500` 毫秒

### 3. 导出的数据结构

生成的代码中，这些字段都是 `long` 类型（毫秒值）：

#### C# 示例
```csharp
public class TaskConfig
{
    public int Id { get; set; }
    public string Name { get; set; }
    public long CooldownTime { get; set; }  // 毫秒
    public long Duration { get; set; }      // 毫秒
    public long ExpireTime { get; set; }    // 毫秒
    public long Delay { get; set; }         // 毫秒
}
```

#### TypeScript 示例
```typescript
export class TaskConfig {
    id: number;
    name: string;
    cooldownTime: number;  // 毫秒
    duration: number;      // 毫秒
    expireTime: number;    // 毫秒
    delay: number;         // 毫秒
}
```

### 4. 在代码中使用

由于导出的值都是毫秒，可以直接用于计时器或时间计算：

#### C# 示例
```csharp
var config = TaskConfigTable.Get(1);

// 设置冷却计时器
var cooldownTimer = new Timer(config.CooldownTime);

// 计算过期时间
var expireDateTime = DateTime.Now.AddMilliseconds(config.ExpireTime);

// 延迟执行
await Task.Delay((int)config.Delay);
```

#### Unity C# 示例
```csharp
var config = TaskConfigTable.Get(1);

// 使用协程延迟
yield return new WaitForSeconds(config.CooldownTime / 1000f);

// 计算剩余时间（秒）
float remainingSeconds = config.Duration / 1000f;
```

#### TypeScript/JavaScript 示例
```typescript
const config = TaskConfigTable.get(1);

// 设置定时器
setTimeout(() => {
    console.log('冷却结束');
}, config.cooldownTime);

// 计算过期时间
const expireDate = new Date(Date.now() + config.expireTime);

// 转换为秒显示
const durationInSeconds = config.duration / 1000;
console.log(`持续时间: ${durationInSeconds} 秒`);
```

## 使用场景

### 1. 游戏技能配置
```xml
<bean name="SkillConfig">
    <var name="id" type="int"/>
    <var name="name" type="string"/>
    <var name="cooldown" type="second"/>        <!-- 技能冷却时间 -->
    <var name="duration" type="second"/>        <!-- 技能持续时间 -->
    <var name="cast_time" type="millisecond"/>  <!-- 施法时间 -->
</bean>
```

### 2. 活动配置
```xml
<bean name="ActivityConfig">
    <var name="id" type="int"/>
    <var name="name" type="string"/>
    <var name="duration" type="day"/>           <!-- 活动持续天数 -->
    <var name="daily_reset" type="hour"/>       <!-- 每日重置时间（小时） -->

```

### 3. Buff 配置

```xml
<bean name="BuffConfig">
    <var name="id" type="int"/>
    <var name="name" type="string"/>
    <var name="duration" type="second"/>        <!-- Buff持续时间 -->
    <var name="tick_interval" type="millisecond"/> <!-- 跳字间隔 -->
</bean>
```

### 4. 任务配置
```xml

    <var name="id" type="int"/>
    <var name="name" type="string"/>
    <var name="time_limit" type="minute"/>      <!-- 任务时间限制 -->
    <var name="expire_days" type="day"/>        <!-- 任务过期天数 -->
</bean>
```

## 注意事项

1. **存储格式**：所有时间类型最终都存储为 `long` 型毫秒值
2. **精度**：毫秒级精度，适用于大多数游戏场景
3. **范围**：long 型可以表示约 292,471,208 年的时间范围，完全满足游戏需求
4. **可空性**：支持可空类型，如 `day?` 表示可空的天数类型
5. **索引键**：这些类型可以作为表的索引键使用（除 `datetime` 外）

## 与 datetime 类型的区别

| 特性 | datetime | day/hour/minute/second/millisecond |
|------|----------|-----------------------------------|
| 输入格式 | 日期时间字符串（如 "2025-1-1 12:00:00"） | 数值 |
| 存储格式 | DateTime 对象 | long 型毫秒值 |
| 用途 | 表示具体的日期时间点 | 表示时间长度/持续时间 |
| 可作为索引键 | 否 | 是 |

## 示例：完整的技能配置

### 定义文件（skill.xml）
```xml
<bean name="SkillConfig">
    <var name="id" type="int"/>
    <var name="name" type="string"/>
    <var name="cooldown" type="second" comment="冷却时间（秒）"/>
    <var name="duration" type="second" comment="持续时间（秒）"/>
    <var name="cast_time" type="millisecond" comment="施法时间（毫秒）"/>
    <var name="channel_tick" type="millisecond" comment="引导技能跳字间隔（毫秒）"/>
</bean>

<table name="TbSkillConfig" value="SkillConfig" input="skill.xlsx"/>
```

### Excel 配置（skill.xlsx）

| id | name | cooldown | duration | cast_time | channel_tick |
|----|------|----------|----------|-----------|--------------|
| 1001 | 火球术 | 5 | 0 | 800 | 0 |
| 1002 | 冰冻术 | 8 | 3 | 1000 | 0 |
| 1003 | 暴风雪 | 30 | 5 | 1500 | 500 |

### 生成的代码使用
```csharp
// 获取技能配置
var skill = TbSkillConfig.Get(1003);

// 技能冷却：30秒 = 30000毫秒
Debug.Log($"冷却时间: {skill.Cooldown}ms");  // 输出: 30000

// 技能持续：5秒 = 5000毫秒
Debug.Log($"持续时间: {skill.Duration}ms");  // 输出: 5000

// 施法时间：1500毫秒
Debug.Log($"施法时间: {skill.CastTime}ms");  // 输出: 1500

// 引导跳字：500毫秒
Debug.Log($"跳字间隔: {skill.ChannelTick}ms");  // 输出: 500
```

## 总结

新增的时间类型让配置更加直观和易读：
- ✅ 策划可以直接填写易懂的数值（如 30 秒，而不是 30000 毫秒）
- ✅ 减少配置错误（不需要手动计算毫秒值）
- ✅ 提高可维护性（一眼就能看出是多少秒/分钟/天）
- ✅ 统一存储格式（都是毫秒 long 型，方便程序使用）
